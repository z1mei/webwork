# 暴龙护卫队 (T-Rex Vanguard) 网页使用与设计文档

## 1. 概述项目
* **项目名称**：暴龙护卫队 (T-Rex Vanguard) —— 团队专属风采展示平台
* **项目需求**：设计并开发一个响应式网页，集中展示团队的整体风貌（霸气侧漏又带点中二的幽默感）以及每位组员的个人背景、专业技能和兴趣爱好，并创新性地加入交互性强的专属迷你游戏，促进外部对团队的了解。
* **组员分工**：
  * **张鹏晖 (项目大管家)**：项目概述、总结展望、项目功能统筹与文档编写。
  * **梁砾俊 (硬核玩家)**：UI视觉设计、功能测试。
  * **曾英棋 (AI驯服者)**：技术支持与项目创意构思。

## 2. 功能模块
* **核心目标**：提供一个高颜值、交互流畅、富有娱乐精神的团队及个人信息聚合展示窗口。
* **主要功能模块**：
  * **全景展示模块**：展示团队名称、打字机特效呈现的团队口号以及核心理念。
  * **成员导航模块**：以 3D 翻转卡片形式提供每位组员的直观入口。
  * **个人详情模块**：图文并茂的弹窗（Modal）展示单人详细信息，包含基础信息、能力雷达进度条、爱好胶囊标签。
  * **互动娱乐区 (Arcade Zone)**：提供三款为成员量身定做的小游戏（暴龙咆哮、硬核特训、提瓦特记忆），并附带本地排行榜功能（TOP 3）。
  * **网页使用文档展示**：内嵌模态框，直观说明网站交互方式。

## 3. 网站架构设计 (Sitemap)
网站采用单页面应用（SPA-like）锚点滑动扁平化结构，确保用户操作简捷。
* **首页 (Home)**
  * **Header (导航栏)**：包含 Logo、各区块平滑跳转锚点、使用文档入口、深浅色模式切换按钮。
  * **Hero Section (团队首屏)**：全屏渐变背景 + 打字机特效 + 滚动引导箭头。
  * **About (团队简介)**：团队目标、愿景及中二释义。
  * **Members (成员名录)**：3张成员翻转卡片网格。
  * **Arcade (互动娱乐区)**：三个迷你游戏入口卡片。
  * **Footer (页脚)**：版权信息及隐藏彩蛋入口。
* **弹窗层 (Modals)**
  * 3个独立的个人详情页面模态框。
  * 3个独立的小游戏试炼模态框（含游戏区及结算排行榜）。
  * 1个网页使用说明模态框。

## 4. UI 设计规范
* **主色调**：**科技蓝** (`#2563eb`)，传达专业与现代科技感。
* **辅助色**：**薄荷绿** (`#10b981`) 用于强调和按钮悬停；**暴龙红** (`#ef4444`) 配合曾英棋的专属模块和游戏。
* **材质风格**：全面采用**毛玻璃特效 (Glassmorphism)**，搭配半透明背景与高斯模糊，展现极简高端的视觉效果。
* **主题系统**：自带一套深色模式 (Dark Mode) 与浅色模式变量，支持无缝切换。
* **字体**：
  * 英文字体：`Inter`。
  * 中文字体：推荐 `Noto Sans SC`（黑体），确保各端阅读的高辨识度。

## 5. 页面的详细设计方案
* **首屏视觉 (Hero Section)**：设置高度为 `100vh`，背景采用流动的对角渐变色。中央使用 JS 配合 CSS 实现打字机闪烁光标特效展示“We are T-Rex Vanguard.”。
* **成员卡片 (Member Grid)**：
  * 使用 CSS Grid 布局构建响应式网格。
  * 利用 `transform: rotateY(180deg)` 实现 3D 翻转动画。默认显示头像和名字，悬停 (Hover) 后翻转显示名言和按钮。
* **成员详情与技能条**：在弹窗展开时，利用 JS 延迟触发 CSS 动画，使技能条从 0% 平滑拉伸至预设宽度（如 90%）。
* **小游戏模块**：
  * 均利用 `position: absolute` 等绝对定位及动态 DOM 生成技术实现。
  * 结算时将分数数组读取/写入浏览器的 `localStorage`，从而实现轻量级的历史 TOP 3 排名榜单。

## 6. 测试与错误调整机制
* **测试维度**：
  * **响应式测试**：在开发者工具中模拟 PC、iPad、iPhone 端尺寸。
  * **交互与内存泄漏测试**：反复开启、关闭游戏弹窗，检查定时器是否正常运作。
* **出错调整方案 (Bug 修复)**：
  * **排版/样式错乱**：通常出现在移动端。调整方案为通过检查工具定位元素，修改 `style.css` 底部 `@media (max-width: 768px)` 媒体查询里的 Flex 布局属性（如从横向改为竖向排列）。
  * **游戏定时器重叠**：若玩家快速多次点击“重新开始”，可能导致倒计时加速（多个 setInterval 冲突）。调整方案是在每次触发 `startGame()` 函数前，强制调用 `clearInterval()` 和 `clearTimeout()` 清理上一个游戏进程的定时器，确保单线程运行。

## 7. 总结和展望
* **项目总结**：本项目成功将前期的策划框架落地为了高完成度的前端展示网页，不仅利用 HTML/CSS/JS 实现了响应式和高颜值设计，还通过互动小游戏极大丰富了趣味性，是一次完整的前端能力实践。
* **未来拓展 (Outlook)**：
  * **数据云端化**：目前数据写死在前端代码或 localStorage 中。未来可引入简单的后端 API 或 Headless CMS，允许非技术组员直接在后台编辑自己的履历内容。
  * **团队项目库**：新增“项目案例”版块，用于存放小组未来合作的其他作业成果或开源项目链接。
* **趣味小工具亮点**：
  * **天选发言人**：右下角的骰子悬浮球，支持一键闪烁并随机抽取一位组员，实乃小组开会抽签利器！
  * **动态彩蛋**：连击最底部的“暴龙护卫队” Logo 5次，将触发震撼全屏的“暴龙战士咆哮”隐藏特效与音效。
