# 《Web前端开发》期末设计作业：暴龙护卫队 (T-Rex Vanguard) 设计文档

## 1. 项目概述

### 1.1 项目背景
在当今互联网时代，一个优秀的团队需要一个能够全方位展示风采与凝聚力的窗口。本次《Web前端开发》课程期末设计作业，要求构建一个功能完整、设计精良的团队介绍网站。我们组建了“暴龙护卫队 (T-Rex Vanguard)”，抛弃了传统的网页模板，从零开始打造了一个基于“极简暗黑高级风（Apple-style Dark Mode）”的团队主页，并在其中融入了三款团队专属的互动迷你游戏（Arcade），力求展现前沿的前端技术与极客精神。

### 1.2 团队名称
**暴龙护卫队 (T-Rex Vanguard)** —— 象征着团队犹如宇宙暴龙般强悍的技术执行力与无畏的探索先锋精神。

### 1.3 成员分工
本组共三名核心成员，分工如下：
- **张鹏晖（项目大管家 / 内容与架构统筹）**：负责项目概述、功能需求统筹、文档结构梳理与最终《设计文档》的撰写。规划并撰写了全站的文案框架。
- **梁砾俊（硬核玩家 / UI设计师与测试）**：负责全站视觉规范制定（暗黑高级风）、毛玻璃特效（Glassmorphism）与动画曲线调试。负责浏览器兼容性测试、排版除错与游戏手感调优。
- **曾英棋（AI驯服者 / 核心开发与创意技术支持）**：负责底层框架搭建、Fluid Typography（流体排版）自适应缩放机制、以及 Arcade 游乐区（三款小游戏）的核心 JavaScript 逻辑开发与创意构思。

---

## 2. 需求分析

为实现兼具美感与互动的展示平台，我们明确了以下功能需求：
1. **团队品牌展示**：首屏需具备极强视觉冲击力，利用动态打字机特效展示 Slogan，凸显团队文化。
2. **成员独立名片**：每位成员拥有独立展示区块，采用左图右文（及交错）的现代化排版，点击后可弹出全屏毛玻璃模态框（Modal）展示详细个人生活、特长与兴趣。
3. **互动游乐区 (Arcade)**：内嵌至少三款原创迷你游戏（暴龙咆哮、硬核特训、提瓦特记忆），需包含完整的数据计算、得分反馈机制。
4. **全端自适应响应**：要求从移动端（手机）到超宽屏显示器（Windows 1080P/2K）均能实现完美缩放，杜绝大黑边和排版错位。

## 3. 信息架构设计

网站采用单页应用（SPA）平滑滚动架构。

### 3.1 网站结构图
```text
暴龙护卫队 主页 (index.html)
├── 顶部导航栏 (Navbar：毛玻璃悬浮导航)
├── 首屏形象 (Hero Section：打字机 Slogan)
├── 成员介绍 (Team Section：交错式排版)
│   ├── 曾英棋 (详情弹窗 Modal-1)
│   ├── 梁砾俊 (详情弹窗 Modal-2)
│   └── 张鹏晖 (详情弹窗 Modal-3)
├── 互动游乐区 (Arcade Section)
│   ├── Game 1: 暴龙咆哮 (解压点击)
│   ├── Game 2: 硬核特训 (反应打靶)
│   └── Game 3: 提瓦特记忆 (翻牌配对)
└── 底部版权与彩蛋区 (Footer)
```

### 3.2 页面流程图
1. **进入网站** -> 浏览全屏暗黑系首屏，观看流体打字文字加载。
2. **浏览成员** -> 滚动至成员区，欣赏左图右文（及右图左文）的交错式对称排版。文字伴随滚动延迟动画（Reveal Delay）优雅淡入。
3. **深入了解** -> 点击成员模块的“深入了解”按钮，弹出全屏毛玻璃 Modal，阅读成员长篇故事与详细技能雷达图。关闭弹窗返回。
4. **体验游戏** -> 滚动至 Arcade 游戏区，点击对应游戏卡片弹出毛玻璃游戏舱，游玩并获得游戏得分反馈。

## 4. 视觉设计规范

本项目全面对标 Apple 官网的现代极简主义设计，摒弃了所有花哨的背景，采用纯粹的暗黑模式。

- **色彩系统**：
  - **主背景色**：`#000000` (纯黑)，提供深邃的沉浸感。
  - **模块辅助色**：`#1c1c1e` (深空灰) 及 `#1d1d1f`，用于区分容器层级。
  - **文本色**：`#f5f5f7` (主标题白)、`#86868b` (副标题灰)。
  - **高亮渐变色**：为每位成员定制了专属渐变色（暴龙红、深海蓝、薄荷绿），用于大标题强调。
- **字体规范**：采用无衬线系统栈 `font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif;`，通过极粗（700）和大号字重凸显标题，细字重保持正文呼吸感。
- **UI设计特色**：大量运用 **Glassmorphism (毛玻璃)** `backdrop-filter: blur(20px)` 替代传统纯色背景，彻底消除传统硬边框。

## 5. 页面详细设计

- **首屏 (Hero)**：页面加载时通过 JS 定时器逐字输出打字机闪烁光标特效展示“We are T-Rex Vanguard”。摒弃无用的边距，使用 `100dvh` 完全释放屏幕高度。
- **成员区 (Team)**：
  - **交错排版**：通过 `flex-direction: row` 和 `row-reverse` 实现“图-文”与“文-图”的交错对称排列，视觉重心不断游走，打破单调。
  - **版心优化**：为解决宽屏显示器黑边过大的问题，将内容最大宽度（`max-width`）设定为 `1440px`，充分利用大屏视野。
  - **流体动画**：引入 Intersection Observer，实现文字和图片的滚动淡入（Fade In Up）和阶梯式延迟（Delay）。
- **Arcade 游乐区**：采用 Flex 弹性布局，三张深色卡片水平等距铺开。鼠标悬浮时触发 `transform: translateY(-5px)` 的微上浮交互。

## 6. 技术实现方案

- **Fluid Typography (流体排版)**：
  利用 CSS 的 `vw` (视口宽度) 单位动态计算字体大小，使页面文字和卡片尺寸随屏幕无级缩放，保证在任意比例下排版均不折行、不错位。
- **单页模态框 (Modal) 管理逻辑**：
  不使用多页面跳转，而是通过动态添加和移除 `.active` 类来控制毛玻璃弹窗的显隐，同时通过 `document.body.style.overflow = 'hidden'` 防止弹窗滚动时底层页面穿透。
- **小游戏核心逻辑**：
  纯原生 JavaScript 实现。如“提瓦特记忆”采用了原生的数组洗牌逻辑生成卡片顺序，“硬核特训”通过定时器随机生成点击坐标。所有游戏共享全局的开关控制函数，确保游戏窗口关闭时清除定时器，避免内存泄漏。

## 7. 测试与完善

- **跨屏兼容性测试**：
  - **问题记录**：在 Windows 1080P/2K 屏幕测试时，发现原先 980px 的限宽导致页面主体太小，左右出现巨大黑边。
  - **解决方案**：重构了 `.nav-container` 和 `.section-content` 的 CSS，将 `max-width` 提升至 `1440px`，并引入 90% 宽度居中对齐，实现了大屏完美适配。
- **移动端底部遮挡测试**：
  - **问题记录**：手机浏览器底部的地址栏和工具栏会导致 `100vh` 产生不必要的滚动条。
  - **解决方案**：引入了较新的 `height: 100dvh`（动态视口高度）属性，完美解决了移动端的首屏遮挡问题。
- **游戏 Bug 修复反馈**：
  - **问题记录**：暴龙咆哮游戏中，连续点击暴龙模型会导致模型尺寸瞬间跳变（缩放割裂）。
  - **解决方案**：引入 CSS 变量 `transform: scale(var(--trex-scale))`，在 JS 中基于当前体积动态计算相对缩放比例，修复了点击动画的割裂感。

## 8. 项目总结与展望

- **团队协作经验**：
  本次开发采用了敏捷协作模式。从确立暗黑风格基调，到分配功能模块，再到整合代码，成员们各司其职。通过反复的“设计->测试->打回重构”循环，有效解决了排版对称性问题和游戏逻辑漏洞。
- **技术难点与突破**：
  在不使用 Vue/React 等框架的前提下，纯手写原生 JavaScript 进行 DOM 操作和复杂状态管理（尤其是多模态框管理和定时器清理）是一大挑战。我们最终通过规范的函数封装和事件监听攻克了这些难点。
- **个人收获及展望**：
  团队成员不仅掌握了现代 Web 高级排版技巧（CSS Flexbox、流体排版、毛玻璃特效），更深刻理解了用户交互体验（UX）的细节打磨。未来，我们计划引入真正的后端 API，将 Arcade 游戏的本地 localStorage 数据接入云端数据库，实现真正的全服实时竞技展示。
