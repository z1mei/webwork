# 《Web前端开发》期末设计作业：暴龙护卫队 (T-Rex Vanguard) 设计文档

## 1. 项目概述

### 1.1 项目背景
在当今互联网时代，一个优秀的团队需要一个能够全方位展示风采与凝聚力的窗口。本次《Web前端开发》课程期末设计作业，要求构建一个功能完整、设计精良的团队介绍网站。我们组建了“暴龙护卫队 (T-Rex Vanguard)”，抛弃了传统的网页模板，从零开始打造了一个基于“极简暗黑高级风（Apple-style Dark Mode）”的团队主页，并在其中融入了三款团队专属的互动迷你游戏（Arcade），力求展现前沿的前端技术与极客精神。

### 1.2 团队名称
**暴龙护卫队 (T-Rex Vanguard)** —— 象征着团队犹如宇宙暴龙般强悍的技术执行力与无畏的探索先锋精神。

### 1.3 成员分工
本组共三名核心成员，分工如下：
- **张鹏晖（项目大管家 / 内容与架构统筹）**：负责项目概述、功能需求统筹、文档结构梳理与最终《设计文档》的撰写。规划并撰写了全站的文案框架。
- **梁砾俊（硬核玩家 / UI设计师与测试）**：负责全站视觉规范制定（暗黑高级风）、卡片毛玻璃特效（Glassmorphism）与动画曲线调试。负责浏览器兼容性测试、排版除错与游戏手感调优。
- **曾英棋（AI驯服者 / 核心开发与创意技术支持）**：负责底层框架搭建、Fluid Typography（流体排版）自适应缩放机制、以及 Arcade 游乐区（三款小游戏）的核心 JavaScript 逻辑开发与创意构思。

## 2. 需求分析

为实现兼具美感与互动的展示平台，我们明确了以下功能需求：
1. **团队品牌展示**：首屏需具备极强视觉冲击力，利用动态打字机特效展示 Slogan，凸显团队文化。
2. **成员独立名片**：每位成员拥有独立展示区块，支持 3D 翻转卡片特效，点击后可弹出全屏毛玻璃模态框（Modal）展示详细个人生活、特长与兴趣。
3. **互动游乐区 (Arcade)**：内嵌至少三款原创迷你游戏（暴龙咆哮、硬核特训、提瓦特记忆），需包含完整的数据计算、得分反馈机制。
4. **全端自适应响应**：要求从移动端（手机）到超宽屏显示器（Windows 1080P/2K）均能实现完美缩放，杜绝大黑边和排版错位。

## 3. 信息架构设计

网站采用单页应用（SPA）平滑滚动架构。

### 3.1 网站结构图
```text
暴龙护卫队 主页 (index.html)
├── 顶部导航栏 (Navbar：毛玻璃悬浮导航)
├── 首屏形象 (Hero Section：打字机 Slogan)
├── 互动游乐区 (Arcade Section)
│   ├── Game 1: 暴龙咆哮 (解压点击)
│   ├── Game 2: 硬核特训 (反应打靶)
│   └── Game 3: 提瓦特记忆 (翻牌配对)
├── 成员介绍 (Team Section：交错式排版与3D卡片)
│   ├── 曾英棋 (详情弹窗 Modal-1)
│   ├── 梁砾俊 (详情弹窗 Modal-2)
│   └── 张鹏晖 (详情弹窗 Modal-3)
└── 底部版权与彩蛋区 (Footer)
```

### 3.2 页面流程图
1. **进入网站** -> 浏览全屏暗黑系首屏，观看流体打字文字加载。
2. **滚动页面** -> 卡片伴随滚动延迟动画（Reveal Delay）进入视野，进入 Arcade 游戏区。
3. **体验游戏** -> 点击对应游戏卡片弹出毛玻璃游戏舱，游玩并获得游戏反馈（如绿色发光特效）。关闭弹窗返回。
4. **浏览成员** -> 滚动至成员区，欣赏左图右文（及右图左文）的交错式对称排版。
5. **深入了解** -> 鼠标悬停卡片触发 3D 翻转，点击“了解更多”弹出全屏 Modal，阅读成员长篇故事。

## 4. 视觉设计规范

本项目全面对标 Apple 官网的现代极简主义设计，摒弃了所有花哨的背景，采用纯粹的暗黑模式。

- **色彩系统**：
  - **主背景色**：`#000000` (纯黑)，提供深邃的沉浸感。
  - **模块辅助色**：`#1c1c1e` (深空灰)，用于卡片与容器背景，形成层级对比。
  - **文本色**：`#f5f5f7` (主标题白)、`#86868b` (副标题灰)。
  - **高亮功能色**：科技蓝 (`#3b82f6`)、暴龙红 (`#ef4444`)、薄荷绿 (`#10b981`)，用于图标点缀与成功/警告反馈。
- **字体规范**：弃用默认宋体，采用无衬线系统栈 `font-family: -apple-system, "SF Pro Text", "Helvetica Neue", sans-serif;`，通过极粗（700）和大号字重凸显标题，细字重保持正文呼吸感。
- **UI设计特色**：大量运用 **Glassmorphism (毛玻璃)** `backdrop-filter: blur(20px)` 及微悬浮发光阴影，彻底消除传统边框。

## 5. 页面详细设计

- **首屏 (Hero)**：页面加载时通过 JS 定时器逐字输出“We are T-Rex Vanguard”，结合光标闪烁动画。摒弃无用的边距，完全释放屏幕宽度。
- **Arcade 游乐区**：采用 Flex 弹性盒子布局，三张卡片水平等距铺开。鼠标悬浮时卡片上浮并投射对应主题色的发光阴影。
- **成员区 (Team)**：
  - **交错排版**：通过 `flex-direction: row` 和 `row-reverse` 实现“图-文”与“文-图”的对称交错排列。
  - **版心优化**：为解决宽屏显示器黑边过大的问题，版心最大宽度（`max-width`）设定为 `1440px`，充分利用高分屏视野。
  - **3D卡片**：运用 `transform-style: preserve-3d;` 和 `backface-visibility: hidden;` 实现 180 度翻转交互。

## 6. 技术实现方案

- **Fluid Typography (流体排版)**：
  利用 CSS 的 `vw` (视口宽度) 单位动态计算字体大小（如 `font-size: clamp(2.5rem, 5vw, 5rem);`），使页面文字和卡片尺寸随屏幕无级缩放。
- **提瓦特记忆（翻牌配对）核心逻辑**：
  使用 Fisher-Yates 算法洗牌，通过数组维护已翻开卡片状态。利用 `setTimeout` 处理翻牌错误时的延迟翻回动作，同时在每次 `startGame` 时强制调用 `clearTimeout()`，防止玩家狂点导致定时器叠加内存溢出。
- **移动端视口适配**：
  使用较新的 `height: 100dvh` 属性替代传统的 `100vh`，完美解决了 iOS/Android 浏览器由于底部地址栏收缩导致的滚动条遮挡问题。

## 7. 测试与完善

- **跨屏兼容性测试**：
  - **问题记录**：在 Windows 1080P/2K 屏幕测试时，发现原先 980px 的限宽导致页面主体太小，左右出现巨大黑边。
  - **解决方案**：重构了 `.nav-container` 和 `.section-content` 的 CSS，将 `max-width` 提升至 `1440px`，并引入 90% 宽度居中对齐，实现了大屏完美适配。
- **游戏 Bug 修复反馈**：
  - **问题记录**：暴龙咆哮游戏中，连续点击暴龙模型会导致模型尺寸瞬间跳变（缩放割裂）。
  - **解决方案**：引入 CSS 变量 `transform: scale(var(--trex-scale))`，在 JS 中基于当前体积动态计算相对缩放比例，修复了点击动画的割裂感。

## 8. 项目总结与展望

- **团队协作经验**：
  本次开发采用了敏捷协作模式。从确立暗黑风格基调，到分配功能模块，再到整合代码，成员们各司其职。通过反复的“设计->测试->打回重构”循环，有效解决了排版对称性问题和游戏逻辑漏洞。
- **技术难点与突破**：
  在不使用 Vue/React 等框架的前提下，纯手写原生 JavaScript 进行 DOM 操作和复杂状态管理（尤其是 Arcade 的多窗口和定时器管理）是一大挑战。我们最终通过闭包封装和事件委托机制攻克了这些难点。
- **个人收获及展望**：
  团队成员不仅掌握了现代 Web 高级排版技巧（CSS Grid/Flexbox、毛玻璃特效），更深刻理解了用户交互体验（UX）的细节打磨。未来，我们计划引入 Node.js 后端，将 Arcade 游戏的排行榜接入云端数据库，实现真正的全服实时竞技展示。
