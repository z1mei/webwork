# 《Web前端开发》期末设计作业：暴龙护卫队 (T-Rex Vanguard) 设计文档

## 一、项目概述

### 1.1 项目背景
在当今互联网时代，一个优秀的团队需要一个能够全方位展示风采与凝聚力的窗口。本次《Web前端开发》课程期末设计作业，要求构建一个功能完整、设计精良的团队介绍网站。我们组建了“暴龙护卫队 (T-Rex Vanguard)”，抛弃了传统的网页模板，从零开始打造了一个基于“极简暗黑高级风（Apple-style Dark Mode）”的团队主页，力求展现前沿的前端技术与团队协作精神。

### 1.2 团队名称
**暴龙护卫队 (T-Rex Vanguard)** —— 象征着团队犹如宇宙暴龙般强悍的技术执行力与无畏的探索先锋精神。

### 1.3 成员结构与详细分工
本组共三名核心成员，涵盖了从项目统筹、UI设计到前端开发的完整闭环协作：
- **张鹏晖（项目大管家 / 内容与架构统筹）**
  - **角色定位**：项目经理/产品经理
  - **具体贡献**：统筹整个项目的开发进度，负责功能需求分析与文档结构梳理。撰写了全站的文案框架，协调团队内部的分工协作，并主笔《设计文档》及《分工说明》。同时负责统筹页面的协作风采展示模块。
- **梁砾俊（硬核玩家 / UI设计师与前端开发）**
  - **角色定位**：UI设计师 / 前端开发工程师
  - **具体贡献**：制定全站视觉设计规范（暗黑高级风），绘制页面原型图。负责毛玻璃特效（Glassmorphism）与动画效果的 CSS 编码实现。深度参与前端页面的排版除错、浏览器跨设备兼容性测试以及游戏手感调优。
- **曾英棋（AI驯服者 / 核心前端开发与性能优化）**
  - **角色定位**：主前端开发工程师
  - **具体贡献**：负责网站首页与底层框架搭建、Fluid Typography（流体排版）自适应缩放机制的研发。独立完成成员个人页的动态交互逻辑，主导开发了互动游乐区（三款小游戏）的核心 JavaScript 逻辑，并对全站代码性能进行了深度优化。

## 二、需求分析

为实现兼具美感与互动的展示平台，严格贴合作业开发要求，我们明确了以下四大核心功能需求：

1. **团队整体形象展示**：
   - 首屏需具备极强视觉冲击力，利用动态打字机特效展示 Slogan，全局彰显团队极简且专业的文化底蕴。
2. **成员个人主页展示（独立区块频道化）**：
   - 每位成员拥有独立的展示区块，点击后弹出全屏沉浸式的毛玻璃个人主页。
   - 个人展示内部分频道，详细涵盖成员的**专业学习、生活日常、兴趣特长以及好物/书籍/电影推荐分享**。
3. **团队协作风采展示**：
   - 在页面中自然融入团队的协作理念与精神，通过精妙的交错排版图文并茂地展现团队的默契配合与分工互补。
4. **工具与互动功能**：
   - 内嵌三款原创迷你互动工具/游戏（暴龙咆哮、硬核特训、提瓦特记忆），提供丰富有趣的用户留存互动体验。

## 三、信息架构设计

网站采用单页应用（SPA）平滑滚动架构，结构逻辑清晰，层级流转自然。

### 3.1 网站结构图
```text
暴龙护卫队 主页 (index.html)
├── 顶部导航栏 (Navbar：毛玻璃悬浮导航，包含平滑滚动锚点)
├── 首屏形象区 (Hero Section：打字机 Slogan与团队愿景)
├── 成员介绍区 (Team Section：交错式排版展示团队协作风采)
│   ├── 曾英棋独立页 (包含专业学习/生活/兴趣/推荐分享)
│   ├── 梁砾俊独立页 (包含专业学习/生活/兴趣/推荐分享)
│   └── 张鹏晖独立页 (包含专业学习/生活/兴趣/推荐分享)
├── 互动游乐区 (Arcade Section：互动小工具)
│   ├── 暴龙咆哮 (解压点击器测试)
│   ├── 硬核特训 (反应打靶训练工具)
│   └── 提瓦特记忆 (翻牌配对记忆力训练)
```

### 3.2 页面流程图
1. **进入网站** -> 浏览全屏暗黑系首屏，感受团队整体形象，观看流体打字加载动画。
2. **浏览成员与协作风采** -> 向下滚动至成员区，欣赏左右交错式对称排版，体会团队结构的互补性与协作精神。文字伴随滚动侦测优雅淡入。
3. **深入了解独立主页** -> 点击任意成员模块的“深入了解”按钮，弹出个人专属独立页，通过分类频道全方位浏览其专业技能、生活剪影与推荐分享。关闭页面返回主线。
4. **体验互动功能** -> 滚动至 Arcade 游戏区，点击工具卡片弹出对应的互动舱，游玩并获得动态数据与得分反馈。

## 四、视觉设计规范

本项目全面对标现代极简主义设计规范，摒弃冗余的背景与边框，采用高级暗黑模式打造纯粹的数字体验。

- **色彩系统**：
  - **主背景色**：`#000000` (纯黑)，为全站提供深邃的基底。
  - **辅助/层级色**：`#1c1c1e` (深空灰) 及 `#1d1d1f`，用于区分不同容器的面层级。
  - **文本与功能色**：`#f5f5f7` (主标题白)、`#86868b` (次级文本灰)，强交互链接使用经典的 Apple Blue (`#2997ff`)。
  - **品牌高亮渐变**：为每位成员深度定制了专属渐变色（如暴龙红、深海蓝、薄荷绿），强化个人视觉标签。
- **字体规范**：
  - 核心采用无衬线系统栈 `font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Helvetica, Arial, sans-serif;`。
  - 标题应用极粗字重（700）形成冲击力，正文保持细字重与较大行高（line-height: 1.6+），大幅提升阅读呼吸感。
- **UI设计理念**：
  - **Glassmorphism (毛玻璃特效)**：全站大量运用 `backdrop-filter: blur(20px)` 搭配半透明背景，消除传统实体边框，实现通透、具有未来感的高级质感。

## 五、页面详细设计

- **首屏 (Hero)**：
  - **原型与布局**：页面加载时内容居中显示，使用 `100dvh` 完全释放屏幕纵向高度。
  - **页面交互**：通过纯原生 JS 定时器逐字输出打字机闪烁光标特效展示“We are T-Rex Vanguard”，下拉时首屏内容呈现优雅的视差隐去感。
- **成员区与团队协作风采 (Team)**：
  - **原型与布局**：通过 Flexbox 的 `row` 和 `row-reverse` 实现“图-文”与“文-图”的交错排版布局，视觉重心不断游走。最大宽度设定为 `1440px` 以充分适配超宽屏。
  - **页面交互**：引入 Intersection Observer 滚动监听，实现元素随页面进入视口时的淡入（Fade In Up）和阶梯式延迟过渡动画。
- **成员个人页 (独立区块频道化)**：
  - **原型与布局**：采用全屏覆盖的独立卡片式设计，页面内部分别划分并排版了“专业学习”、“生活日常”、“兴趣特长”与“好物推荐”四大频道区块。
  - **页面交互**：点击唤出时触发 CSS 缩放与渐显动画，同时代码自动锁定底层 `body` 的滚动条防止滑动穿透，提供沉浸式的阅读体验。
- **互动游乐与工具区 (Arcade)**：
  - **原型与布局**：采用 Flex 弹性栅格布局，深色卡片水平等距展开。
  - **页面交互**：鼠标悬浮时触发微上浮（`translateY(-5px)`）与发光反馈，点击进入沉浸式的游戏模态框，包含完整的状态机（Start/Play/End）。

## 六、技术实现方案

本项目遵循纯粹的原生前端技术栈（HTML5 + CSS3 + Vanilla JavaScript），未引入重量级第三方框架，彰显团队扎实的底层编码功底。

1. **Fluid Typography (流体排版) 与自适应机制**：
   - 摒弃了僵硬的媒体查询断点（Media Queries），全局利用 CSS 的 `vw` / `rem` 相对单位和 `clamp()` 函数动态计算尺寸。确保从手机到 4K 显示器，页面文字和容器均能实现平滑的无级缩放。
2. **Intersection Observer 视差侦测**：
   - 抛弃了传统消耗性能的 `window.onscroll` 事件，利用原生的 DOM 观测 API 异步监听元素。当元素进入视口一定比例时动态追加激活类，高效触发渲染层动画。
3. **单页组件与路由逻辑闭环**：
   - 将独立个人主页、互动游戏以 DOM 节点形式预渲染，统一通过 `data-target` 属性匹配并在 JS 中抽离出全局的 Modal 管理器，通过增删 `active` 类名实现视图的切换与生命周期管理。
4. **小游戏与工具核心逻辑**：
   - “提瓦特记忆”采用 Fisher-Yates 洗牌算法生成乱序卡片矩阵，利用事件委托高效处理点击与卡牌配对校验。
   - 针对闭包中的倒计时重叠内存问题，在全局弹窗关闭回调中严格清空了所有关联的 `Timeout` 和 `Interval` 定时器。

## 七、测试与完善

- **浏览器兼容性与路径访问测试**：
  - **问题记录**：早期版本在 Windows 系统的极高分辨率屏幕下，由于宽度硬编码导致左右黑边巨大。而在手机端 Safari 浏览器中，传统的 `100vh` 经常会被底部物理导航栏遮挡，导致访问出现盲区。
  - **解决方案**：团队重构了核心容器层 CSS，全面替换为流体单位，并将首屏高度改写为最新的 `100dvh`（动态视口高度）属性。经多轮测试，在 Chrome、Edge、Firefox、Safari 及各类移动端设备上均实现了近乎完美的跨平台兼容。
- **代码交互与边界问题测试**：
  - **问题记录**：在测试小游戏容器时发现，极端分辨率下卡片翻转效果会被底层布局遮挡。
  - **解决方案**：引入 z-index 分层管理，同时对小游戏模态框宽度引入了自适应机制限制，大幅提升了极端操作情况下的页面鲁棒性。

## 八、项目总结与展望

### 8.1 团队协作经验
在本次漫长而充实的设计开发过程中，我们深刻体会到了合理分工带来的“多核驱动”协作力量。项目经理在第14周精确拆解了《作业开发要求》，明确了各个节点的任务目标；UI设计师提前输出了严谨的视觉与尺寸规范，使得前端开发在编写代码时无需反复猜想和调整样式。在遇到模块样式冲突时，团队成员通过线下讨论与及时的沟通迅速整合代码，展现了极高的配合度。

### 8.2 设计过程中的技术难点与解决方案
**技术难点**：在坚持不使用 Vue/React 等现代化框架的前提下，如何优雅地管理数量众多且复杂的层级结构（3个成员详情页、3个独立交互小工具），而不导致 DOM 事件混乱与代码冗余？
**解决方案**：我们在原生 JavaScript 中抽象了一套类似于组件化思想的“全局视图管理器”。通过统一定义生命周期（打开前、挂载、关闭后），将重复的弹窗锁定机制、动画触发和定时器清理封装进高阶函数中，实现了代码的高内聚与低耦合，极大降低了系统复杂度。

### 8.3 个人收获及未来展望
此次期末作业不仅仅是一次简单的编码练习，更是对我们审美、逻辑、工程化思维的全面锤炼。它让我们深入掌握了现代 CSS 高级布局手法（Flex/Grid/变量系统/滤镜特效）与原生 JS 的 DOM 深度操控机制，同时也让我们学会了如何从零开始、按部就班地规划并落成一个完整的数字媒体产品。

**未来展望**：受限于本次作业的纯前端性质，部分互动体验尚未接入真实的后端数据处理。未来在学习了 Node.js 或是数据库课程后，我们希望为该网站引入真正的后端 RESTful API 服务，甚至进一步将小游戏的数据做成全网在线排行榜，将本项目升华为一个拥有极强生命力的全栈级作品。
