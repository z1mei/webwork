# 《Web前端开发》期末设计作业：暴龙护卫队 (T-Rex Vanguard) 设计文档

## 1. 项目概述

### 1.1 项目背景
在当今互联网时代，一个优秀的团队不仅需要卓越的技术实力，更需要一个能够全方位、立体化展示团队风采与凝聚力的窗口。本次《Web前端开发》课程期末设计作业，要求构建一个功能完整、设计精良的团队介绍网站。基于此要求，我们组建了“暴龙护卫队 (T-Rex Vanguard)”，旨在通过最前沿的 Web 技术与极简的高级视觉美学，打造一个兼具沉浸感与互动性的现代化团队主页，不仅展示成员的专业实力与兴趣特长，更通过丰富的交互体验（如内嵌互动小游戏）传达团队充满活力与极客精神的技术氛围。

### 1.2 团队名称
**暴龙护卫队 (T-Rex Vanguard)** —— 象征着团队在前端开发领域犹如暴龙般强悍的执行力，同时作为先锋（Vanguard），不断探索创新边界。

### 1.3 成员分工
为了保证项目的高效推进与最终的高质量产出，团队进行了明确的职能划分（注：本组共计三名核心成员，均承担了核心开发与设计任务）：
- **曾英棋（项目经理 / 前端开发工程师）**：
  - **统筹进度**：负责项目的整体进度把控，制定开发里程碑。
  - **核心开发**：主导项目基础框架的搭建，完成首页首屏全屏滚动逻辑、打字机特效开发。负责 Arcade 游乐区的核心 JS 逻辑编写，包括小游戏的状态管理与底层通信。
  - **AI 协作核心**：熟练运用 AI 辅助编程工具，解决代码重构及性能优化问题。

- **梁砾俊（UI设计师 / 前端测试工程师）**：
  - **视觉设计**：制定全站的“暗黑高级风（Apple-style Dark Mode）”视觉规范。设计 UI 交互原型，调整卡片毛玻璃（Glassmorphism）质感，优化动画曲线（如平滑的 CSS 3D 翻转过渡）。
  - **质量把控**：负责各模块的浏览器兼容性测试、移动端 100dvh 响应式适配，并深度参与 Arcade 游戏的找虫（Bug Hunting）与交互手感调优。

- **张鹏晖（项目统筹 / 内容策划开发）**：
  - **内容规划**：负责全站文案策划、组员信息梳理及最终《设计文档》的统筹编撰。
  - **个人主页及模块开发**：负责实现团队成员展示模块（Grid布局、Modal 弹窗逻辑），将成员的专业学习、生活兴趣、推荐分享等内容以现代化的组件呈现。负责全站反馈机制（联系方式与 Feedback 弹窗）的开发。

## 2. 需求分析

通过对课程任务书的拆解，我们明确了以下核心功能需求，以确保网站内容丰富且互动性强：

1. **团队整体形象展示**：
   - 需要一个极具视觉冲击力的首屏（Hero Section），能够瞬间抓住用户眼球。
   - 使用动态文字（打字机效果）展示团队的 Slogan 与核心价值观。
2. **成员个人主页展示（区块化展现）**：
   - 为每位成员分配独立的展示卡片，支持悬浮或点击后查看更多信息。
   - 需包含个人照片、角色定位、技术栈特长、生活兴趣爱好及个人座右铭，通过弹窗（Modal）的方式进行扩展阅读。
3. **团队协作与互动功能（Arcade 游乐区）**：
   - 突破传统枯燥的文字介绍，设计一个名为“Arcade”的内置游乐区，包含至少三款互动小游戏（暴龙咆哮、硬核特训、提瓦特记忆）。
   - 小游戏需具备完整的逻辑、得分反馈与重新开始机制，以此展示团队扎实的 JavaScript 编程功底与交互设计能力。
4. **联系方式与用户反馈**：
   - 全局导航栏提供“反馈”入口，点击弹出表单，允许用户填写意见并提交，增强网站的互动性与完整性。

## 3. 信息架构设计

为了保证用户体验的流畅性，网站采用了单页应用（SPA）式的平滑滚动架构（Smooth Scroll Section Layout）。

### 3.1 网站结构图
```text
暴龙护卫队 主页 (index.html)
├── 顶部导航栏 (Navbar)
│   ├── Logo / 团队名称
│   ├── 锚点导航 (首屏 / 团队 / 互动 / 反馈)
├── 首屏形象 (Hero Section)
│   ├── 动态打字机 Slogan
│   └── 下拉引导箭头
├── 成员介绍 (Team Section)
│   ├── 曾英棋 (卡片正反面 -> 详情弹窗 Modal-1)
│   ├── 梁砾俊 (卡片正反面 -> 详情弹窗 Modal-2)
│   └── 张鹏晖 (卡片正反面 -> 详情弹窗 Modal-3)
├── 互动游乐区 (Arcade Section)
│   ├── Game 1: 暴龙咆哮 (点击反馈互动)
│   ├── Game 2: 硬核特训 (准星瞄准游戏)
│   └── Game 3: 提瓦特记忆 (卡牌翻转匹配游戏)
└── 底部反馈 (Footer & Feedback)
    └── 意见反馈表单弹窗
```

### 3.2 页面流程图 (User Flow)
1. **进入网站** -> 看到首屏全屏暗黑系界面与自动播放的打字机文字。
2. **滚动页面** -> 卡片以错落有致的延迟动画（Reveal Delay）进入视野，展示三位成员。
3. **点击成员卡片** -> 触发 CSS 3D 翻转查看格言，点击“进一步了解” -> 弹出全屏毛玻璃 Modal 显示个人生活、学习兴趣、书籍推荐。
4. **继续滚动** -> 进入 Arcade 区 -> 选择一款游戏（如“提瓦特记忆”），点击开始 -> 游玩并获得分数反馈。
5. **结束浏览** -> 点击顶部“反馈”按钮 -> 填写意见 -> 提交表单完成闭环。

## 4. 视觉设计规范

本项目高度对标现代极简主义设计（如 Apple 官网风格），以深色模式为默认基调。

### 4.1 色彩系统
- **主色（背景色）**：`#000000` (纯黑)，用于全站基础背景，营造沉浸感与高级感。
- **辅助色（卡片/模块背景）**：`#1c1c1e` (深空灰)，用于区分页面层级，使内容区块在黑色背景上自然凸显。
- **文本色**：`#f5f5f7` (主标题白)、`#86868b` (副标题/辅助说明灰)。
- **功能色/高亮色**：
  - **系统蓝 (Apple Link)**：`#2997ff`，用于链接、主要操作按钮。
  - **系统绿 (Success Green)**：`#30D158`，用于游戏匹配成功、操作成功的霓虹发光提示。
  - **系统红 (Error/Alert)**：`#ef4444`，用于警告或彩蛋背景。

### 4.2 字体规范
- 全站优先使用系统级无衬线字体栈：
  `font-family: -apple-system, BlinkMacSystemFont, "SF Pro Text", "PingFang SC", "Helvetica Neue", Arial, sans-serif;`
- **排版原则**：字号拉开层级对比，大标题采用极粗字重（`font-weight: 700/800`），正文采用极细字重以保持呼吸感。

### 4.3 UI设计元素
- **Glassmorphism (毛玻璃)**：在弹窗（Modal）和顶层导航栏使用 `backdrop-filter: blur(20px); background: rgba(0,0,0,0.7);` 营造现代通透感。
- **微交互与过渡**：所有悬浮、点击状态均添加 `transition: all 0.3s ease;`，确保动画流畅不生硬。

## 5. 页面详细设计

### 5.1 首页首屏 (Hero)
- **原型与布局**：占据 100dvh 高度，内容垂直居中。页面左对齐展示超大号的 Glitch / Typewriter 特效文字。
- **交互说明**：页面加载完毕后，文字逐字显现；底部有一个闪烁的向下箭头指示用户滚动。

### 5.2 团队成员页 (Team)
- **布局**：采用 CSS Grid 布局，三张卡片水平等距排列，响应式下（移动端）自动转为垂直堆叠。
- **交互说明**：
  - **卡片翻转**：使用 `transform-style: preserve-3d;` 和 `backface-visibility: hidden;`，鼠标悬停时沿 Y 轴旋转 180 度展示背面。
  - **详情弹窗**：点击背面按钮，JavaScript 操作添加 `.active` 类，触发弹窗遮罩透明度渐变，内部内容平滑上升显现。

### 5.3 互动游乐区 (Arcade)
包含三个精美的方块游戏区块。
- **提瓦特记忆 (Memory Match)**：
  - 界面为 4x4 的卡牌网格，默认显示暗灰色背板。
  - 玩家点击卡片触发 3D 翻转，若两张卡片图案一致，卡片边框亮起 `#30D158` 的绿色内发光；若不一致，延迟 1 秒后自动翻转回去。
- **UI 体验升级与悬浮提示**：
  - 游戏入口卡片增加物理悬浮感（`translateY` 上浮）及半透明发光特效（Box-shadow glow），强化可点击的视觉暗示。
  - 每个游戏弹窗左上角配备“玩法”悬浮提示（纯 CSS 无感交互），鼠标掠过即可呼出毛玻璃质感的操作指南，不干扰游戏主体进程。
- **沉浸式结算面板重构**：
  - 游戏结束后的结算屏幕彻底重构为高级 Flexbox 左右分栏布局（左侧数据显示，右侧排行榜展示）。该设计摒弃了丑陋的滚动条，使得左右视觉重心稳固，大幅提升了“现代数据面板”的高级极客感。

## 6. 技术实现方案

本项目基于原生 HTML5 + CSS3 + Vanilla JavaScript 开发，未引入厚重的第三方框架（除图标库外），充分锻炼了底层代码能力。

### 6.1 打字机特效实现逻辑
通过 JavaScript 定时器逐字截取字符串内容并更新到 DOM 树中。通过监听字数长度判断是否输入完毕，配合 CSS 的光标闪烁动画模拟真实打字效果。

### 6.2 提瓦特记忆（Game 3）核心逻辑
该游戏难点在于状态管理与异步清空。
- 采用数组洗牌算法（Fisher-Yates Shuffle）在每次 `startGame` 时打乱卡牌顺序。
- 维护两个状态变量：`flippedCards`（当前翻开的卡片），`isLocked`（防止玩家在翻转动画期间狂点导致逻辑错乱）。
- 当玩家翻开两张牌时不匹配，使用 `setTimeout` 延迟 1 秒再执行翻回操作，同时此定时器 ID 被全局保存（`g3MismatchTimeout`），如果玩家在此期间强制点击“重新开始”，可通过 `clearTimeout` 彻底销毁残留的倒计时炸弹，避免报错崩溃。

### 6.3 移动端 100vh 高度适配
在 iOS 等移动端浏览器中，底部的导航栏会随着滑动隐藏，导致 `100vh` 计算出现滚动条或被遮挡。我们采用了最新的 CSS 属性 `height: 100dvh;`（动态视口高度）来完美解决此兼容性问题。

## 7. 测试与完善

项目开发进入中后期，我们进行了严格的跨设备测试与底层除虫（Bug Fix）。

### 7.1 测试环境与工具
- **设备**：MacBook Pro, iPhone 14 Pro, Android 测试机。
- **浏览器**：Chrome, Safari, Microsoft Edge。

### 7.2 典型问题记录与解决方案
- **Bug 1：暴龙咆哮游戏的“跳变”体验问题**
  - **现象**：当玩家连续点击暴龙时，暴龙根据得分逐渐变大，但鼠标按下瞬间模型会突然缩小为初始比例，松开又剧烈跳大，视觉体验极速割裂。
  - **解决**：这是由于 CSS 的 `:active` 伪类写死了绝对缩放比例 `transform: scale(0.9)` 覆盖了 JS 动态赋予的放大值。我们通过引入 CSS 变量 `transform: scale(var(--trex-scale, 1))`，并在 JS 中使用 `setProperty('--trex-scale')` 动态传值，完美实现了**在当前体型基础上的等比例相对缩小**。

- **Bug 2：记忆卡片翻转的“透视”Bug**
  - **现象**：卡片在未翻开时，能直接看到背面的图标，破坏了游戏规则。
  - **解决**：排查发现 JS 动态生成的 HTML 结构多包裹了一层 `<div>`，导致 CSS 中的 `preserve-3d` 与 `backface-visibility: hidden` 的上下文关系断裂。通过精简 DOM 结构，严格对齐 CSS 层级，成功修复。

## 8. 项目总结与展望

### 8.1 团队协作经验
本次开发过程中，我们最大的体会是**规范化协作与敏捷响应**的重要性。通过 Git 进行版本控制，我们在 `main` 和 `dev` 分支上进行了多次安全的迭代；期间利用代码审查（Code Review），在提交 PR 时及时发现并合并了样式表冲突问题。团队氛围极佳，各司其职，从设计讨论到代码落地真正做到了一气呵成。

### 8.2 技术难点攻克
在不依赖 React/Vue 的情况下，手动管理 DOM 的状态机极其繁琐，尤其是在 Arcade 三款游戏中。我们学会了如何使用闭包、事件委托机制，以及如何运用 CSS 的 3D 转换（Transform 3D）打造流畅的 GPU 硬件加速动画，这极大地夯实了我们的前端原生底盘。

### 8.3 个人收获与未来展望
- **收获**：掌握了现代 Web 设计美学（暗黑模式、无边界卡片、毛玻璃质感）的代码实现方法，深刻理解了原生 JS 事件循环与内存防泄露机制。
- **展望**：受限于时间，目前的联系我们表单仅做了前端拦截校验。未来计划接入 Node.js/Express 后端环境，实现表单数据的真实存储；同时，计划为 Arcade 游戏引入基于 WebSocket 的全服实时排行榜，让网站彻底突破静态页面的限制，向全栈 Web 应用迈进。
